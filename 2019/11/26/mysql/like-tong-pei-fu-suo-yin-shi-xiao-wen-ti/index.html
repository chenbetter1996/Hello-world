LIKE通配符索引失效问题
MySQL
MySQL
发布日期:   2019-11-26
文章字数:   196
阅读时长:   1 分

问题

因为B+树是叶子节点区间检索。对于匹配串左边，使用通配符肯定无法使用索引。除非SELECT只检索当前索引所属字段。

网上的方法说的用LOCATE(#{keyword}, column)和INSTR(column, #{keyword}),其实是没有用到索引的，只是对字符串截取查找快点而已。

还有说的是REVERSE()函数，把字段和判断值同时反转，但是细想也是不可行的，字段反转但是索引B+树还是之前的排序，依旧用不到

解决

可以创建维护一个索引字段相反的列，对于上面那种是因为revese(title)不是字段问题。创建维护reverse_title字段。虽然麻烦但是简单。
