JavaFX常用总结
JavaFX
Java

1、安装

JavaFX是JavaSE自带的类库，只需要下载可视化插件即可SceneBuilder即可，目前Windows的是直装msi版，只是v2.0，好多年没更新了。另外也可以用OpenJDK的JavaFX的SDK，持续更新。
安装好后，在Eclipse或者IDEA打开fxml文件都可以选择SceneBuilder可视化方式。

几个JavaFX教程网站

  1. JavaFX 教程 （中文）
  2. W3Cschool
  3. 易百教程
  4. 官文档

2、使用

程序运行后与可视化编辑界面有出入，布局容易变化 

<Pane prefHeight="515.0" prefWidth="703.0" xmlns="http://javafx.com/javafx/8" xmlns:fx="http://javafx.com/fxml/1" fx:controller="chgl16.space.RuleController">

...

<TextField fx:id="conditionTextFieldComponent" layoutX="12.0" layoutY="406.0" prefHeight="39.0" prefWidth="320.0" promptText="xx" />

<Button layoutX="564.0" layoutY="349.0" mnemonicParsing="false" onAction="#goReason" prefHeight="64.0" prefWidth="176.0" style="-fx-background-color: #28a745;" text="进行推理" textFill="WHITE">

...


</Pane>

最外层的是Pane，有多种。指定控制类fx:controller=”xxx”。
各种组件添加fx:id唯一标识就可以在fxml文件对应的Controller中映射注入，同名即可。
事件有很多种，最通用的是onAction=”#goReason”，方法无参数。控制器内必须实现。

@FXML
private TextField conditionTextFieldComponent

public void reason() {}

2.1 窗口切换

每个窗口是一个Stage对象。主类默认创建了一个 

 @Override
public void start(Stage primaryStage) throws Exception{
    Parent root = FXMLLoader.load(getClass().getResource("view/index.fxml"));
    primaryStage.setTitle("系统");
    primaryStage.setScene(new Scene(root));
    // 窗口不可伸缩
    primaryStage.setResizable(false);
    primaryStage.show();
}


public static void main(String[] args) {
    launch(args);
}

如果需要程序同一个窗口，只需要每次更换Scene对象即可。每个Scene对应一个FXML文件，作用域是 Stage > Scene > Component

需要额外的推出按钮，只需要stage.close()或者stage.hide()

2.2 样式

样式也是css，但是属性都特定的前缀 -fx ，有些样式和css也不同。
比如文本框内的字体颜色。只能是 

textFieldComponent.setStyle(" -fx-text-fill:RED");

样式可以每个独立设置，也可也设置一些class。也可使用外部文件。跟前端一样 

scene.getStylesheets().add(getClass().getResource("application.css").toExternalForm());

2.3 表格

表格是TableView组件，里面包括TableColumn组件，也就是表格的列标题，数据是和TableColumn绑定的。JavaFX对表格有很多API。

需要打开Scene就加载表格数据，可以使用初始化函数，fxml文件被打开就自动调用 

public class RuleController implements Initializable {
    public void initialize(URL location, ResourceBundle resources) {
        //...
    }
}

表格的数据每个记录都是一个JavaBean，但是这个Bean要是JavaF形式的bean 

@FXML
private TableColumn<RuleProperty, Integer> idTableColumnComponent;

@FXML
private TableColumn<RuleProperty, String> conditionTableColumnComponent;

@FXML
private TableColumn<RuleProperty, String> resultTableColumnComponent;

<RuleProperty, Integer> 的RuleProperty就是这种规范的Bean，Bean里面的属性类型都需要用JavaFX的包装属性类，泛型后面的Integer表示的是这个列的数据的类型。对应里面的SimpleIntegerProperty id。 

public class RuleProperty {
    private SimpleIntegerProperty id;

    private SimpleStringProperty condition;

    private SimpleStringProperty result;

    public RuleProperty(Integer id, List condition, String result) {
        this.id = new SimpleIntegerProperty(id);
        this.condition = new SimpleStringProperty(condition.toString());
        this.result = new SimpleStringProperty(result);
    }

    public Integer getId() {
        return id.get();
    }

    public void setId(Integer id) {
        this.id.set(id);
    }

    public String getCondition() {
        return condition.get();
    }

    public void setCondition(List condition) {
        this.condition.set(condition.toString());
    }

    public String getResult() {
        return result.get();
    }

    public void setResult(String result) {
        this.result.set(result);
    }
}

填充数据

TableView每个记录是一个XxProperty的Bean，因此即便有普通的记录对象Rule了，也需要额外写一个RuleProperty，类实JPA的Entity。全部数据就是一个List，不过JavaFX对这个List也封装要求了，ObservableList 

// 填充表格数据
ObservableList<RuleProperty> data = FXCollections.observableArrayList();
for (int i = 0; i < count; ++i) {
    data.add(new RuleProperty(xx));
}

// 字段数据绑定
// 后面的String泛型是列字段类型，最后的condition是映射到RuleProperty对象的condition字段
conditionTableColumnComponent.setCellValueFactory(new PropertyValueFactory<RuleProperty, String>("condition"));
idTableColumnComponent.setCellValueFactory(new PropertyValueFactory("id"));
resultTableColumnComponent.setCellValueFactory(new PropertyValueFactory<RuleProperty, String>("result"));

// 加入表格展示
tableViewComponent.setItems(data);
  1. ObservableList列表可以通过FXCollections.observableArrayList()构造一个空给的。
  2. TableColumn对象必须指定XxProperty Bean中的某个属性名，即数据绑定映射。使用方法etCellValueFactory(new PropertyValueFactory<RuleProperty, String>(“Bean中属性名”));

    这个泛型必须在@FXML TableComponent xx处声明，不然数据无法解析。

添加记录

// 添加按钮事件绑定
addButtonComponent.setOnAction((ActionEvent e) -> {
    String condition = conditionTextFieldComponent.getText();
    String result = resultTextFieldComponent.getText();

    // 非空、格式判断

    // table添加局部刷新，data是ObservableList<RuleProperty>
    data.add(new RuleProperty(Core.ruleBase.size(), condition, result));

    // 清空输入框
    conditionTextFieldComponent.clear();
    resultTextFieldComponent.clear();
});

组件绑定事件，可以通过fxml中onAction=”#方法名”映射到控制器，也可以在控制器代码中直接对fxid的控件setOnAction()，类似html和js调用事件的关系。

要获取表格的ObservableList记录列表，然后直接添加即可。界面会直接刷新。（内部已经实现了，无需重载Scene或者调用initialize(URL location, ResourceBundle resources) 方法）

删除数据

deleteButtonComponent.setOnAction((ActionEvent e) -> {
    // 获取删除的记录编号
    if (idTextFieldComponent.getText().equals("")) {
        System.err.println("编号不允许为空");
        return;
    }
    Integer id = null;
    try {
        id = Integer.valueOf(idTextFieldComponent.getText());
    } catch (NumberFormatException ex) {
        System.err.println("非法字符");
        return;
    }
    if (id > Core.ruleBase.size() || id <= 0) {
        System.err.println("非法边界");
        return;
    }

    // data.remove(id - 1);
    // 刷新保证不会中断编号
    initialize(location, resources);
    idTextFieldComponent.clear();
});

可以获取到记录编号后直接在ObservableList记录表中remove，但是它删了自动局部刷新，有编号字段会断层。因此最好调用initialize(location, resources全局刷新。

修改数据

// 字段数据可编辑
conditionTableColumnComponent.setCellFactory(TextFieldTableCell.<RuleProperty>forTableColumn());
resultTableColumnComponent.setCellFactory(TextFieldTableCell.<RuleProperty>forTableColumn());

// 编辑提交触发函数
conditionTableColumnComponent.setOnEditCommit(
        (TableColumn.CellEditEvent<RuleProperty, String> t) -> {
            // 修改行数，0起
            int row = t.getTablePosition().getRow();
            // 修改后内容
            String newValue = t.getNewValue();
            System.out.println("修改了第" + (row + 1) + "行体条件，新内容为：" + newValue);

            // 更新到需要的地方
        });
resultTableColumnComponent.setOnEditCommit(
        (TableColumn.CellEditEvent<RuleProperty, String> t) -> {
            // 修改行数，0起
            int row = t.getTablePosition().getRow();
            // 修改后内容
            String newValue = t.getNewValue();
            System.out.println("修改了第" + (row + 1) + "行结果，新内容为：" + newValue);

            // 更新到需要的地方
        });

conditionTableColumnComponent.setCellFactory(TextFieldTableCell.forTableColumn());
单位都是列TableColumn，这样就就可以整列的cell单元格变得可以修改了，可以直接双击修改Enter保存。就一个setCellFactory，刚刚的setCellValueFactory是绑定数据。

但是这样编辑修改保存，只能临时作用于当前界面。虽然修改到了ObservableList，但是ObservableList只是一个中介表，我们需要把修改保存到其他需要的地方。需要一个事件触发的回调函数。

也就是conditionTableColumnComponent.setOnEditCommit(EventHandler<CellEditEvent<S,T>> value);
参数是一个事件，API提供了TableColumn.CellEditEvent<RuleProperty, String> t类，满足函数式编程，事件变量t可以做很多操作 

// 修改行数，0起
int row = t.getTablePosition().getRow();
// 修改后内容
String newValue = t.getNewValue();
// 原来内容
String oldValue = t.getOldValue();

需要注意的是：声明TableColumn的时候必须注明泛型 

@FXML
private TableColumn<RuleProperty, String> conditionTableColumnComponent;

否则报错setOnEditCommit的参数需要是Even而不是EventHandler，其实EventHandler就是Even的子类。原因是事件不能先于所在列先声明泛型。不然private TableColumn conditionTableColumnComponent就是原始类型了。后面要强转了。见StackOverFlow

3. 打包

3.1文件找不到


使用IDEA开发的JavaFX程序，指定好主类后（无需.java后缀）即可在菜单栏Build->Build Artifacts打包

但是打包会有个资源文件路径错乱的情况。打包后jar包的第一层就是src下的包，不包括src目录。类实的如果有文本文件哪怕new File(“filePath”)使用打包后正确的相对路径也没用。打包后已经无法找到文件。 

private static final String RULES_FILE_PATH = "rules.txt";

// 不使用File读取，打包后路径问题无法解决，一般的相对路径也无效。只能Clsss.getResourceAsStream(RULES_FILE_PATH)
// 读取初始化文件
// File file = new File(RULES_FILE_PATH);
String encoding = "UTF-8";
//if (file.isFile() && file.exists()) {
    try {
        //InputStreamReader inputStreamReader = new InputStreamReader(new FileInputStream(file), encoding);
        InputStream inputStream = Core.class.getResourceAsStream(RULES_FILE_PATH);
        BufferedReader bufferedReader = new BufferedReader(new InputStreamReader(inputStream));
        String line = null;
        while ((line = bufferedReader.readLine()) != null) {
            // do
        }
        inputStream.close();
        bufferedReader.close();
    } catch (Exception e) {
        System.err.println("读取初始化文件内容出错");
    }
//} else {
    //  System.out.println("找不到指定文件");
//}

可以把new File(filePath)的方式改为Class.getResourceAsStream(path)，返回一个InputStream。这样运行和打包都没问题。

3.2 fxml文件报错Location is required.

这个是无法读取到fxml文件，路径错误。运行时一般问题不大。都能调对。比如 

// Main.java 类
/* 目录结构
 * controller
 *    - AaControllerNot
 *    - BbController
 * view
 *    - index.fxml
 *    - rule.fmxl
 * Main.java
 *
Parent root = FXMLLoader.load(getClass().getResource("view/index.fxml"));

在Main.java和view目录在一级目录下的时候，调试和打包后执行都没问题。

但是如果是

// AaController.java 类
Parent root = FXMLLoader.load(getClass().getResource("../view/index.fxml"));

分别是一个同级目录下的。这样调试运行也没问题，打包路径也对。但是会报错java.lang.NullPointerException: Location is required. 其实还是想不通。打开Jar包看相对路径确实没问题。

解决：
网上有很多说加入类加载器 getClass().getClassLoader.getResource(“index.fxml”); 或者参数加”/“等等，各种都是不行的。
其实换个想法，Main那个可以，那完全可以使用Main的class，如下 

// AaController.java 类
Parent root = FXMLLoader.load(Main.class.getResource("view/rules.fxml"));

这样调试打包都可以成功

3.3 部分乱码

哪怕都是UTF-8，打包后还是存在file.encoding属性被打包程序更改的问题，解决方法是打包程序运行时指定编码。 

# 运行编码有问题，需指定编码
java -Dfile.encoding=utf-8 -jar xxx.jar

或者在对应乱码的代码处，强制添加编码类型encoding
