同步异步阻塞非阻塞
同步异步 阻塞非阻塞
杂记
发布日期:   2019-05-01
文章字数:   571
阅读时长:   2 分

同步

程序所有操作结束，才返回给用户。用户需要一直等待。而且不能做其他事情。体验很差。但是可以保证执行的顺序。

比如银行的转账系统，需要一直在那个界面等待成功。

异步

两者区别主要是用户不需要等待，异步程序给用户一个反馈表示请求已经开始处理即可。用户可以去干其他的事情。最后出结果再告知用户即可。可以大大提高系统的吞吐量。

比如常用的MQ，先将请求数据放到队列。响应了用户请求后慢慢处理，也有缓存削峰的效果。

又比如常见Ajax交互，默认是异步的。

// 1

$.ajax({
    type: "post",
    url: "",
    async: true,
    data: "",
    contentType: "application/json; charset=utf-8",
    dataType: "json",
    success: function (msg) {
        // 处理
    },
    error: function (error) {
        alert(error.responseText);
    }
});

// 2

这async: true是默认的，即异步。当从1到Ajax时，程序可以继续2操作。而不用阻塞等待进入success或者error方法，这两个回调方法会在结果返回后再调用。

阻塞 & 非阻塞

很容易和同步异步混淆，阻塞和非阻塞主要对同步而言的。非阻塞就是去请求数据服务时，如果还没准备好未成功，也会立即返回结果。重要的是马上有结果，哪怕是未完成的结果。而非阻塞感觉就是没有进度条那种，一直等待期间没有任何结果说明。

比如Ajax同步时，服务器还没有处理完。那么请求一次未果就返回。过段时间再去请求。

比如煮开水的例子。

  1. 一直在那里等到煮开才走就是同步阻塞BIO。
  2. 而发现没煮开走开一段时间（返回未煮开的结果），过段时间再去看。即同步非阻塞NIO。

  3. 如果给煮水器加个煮开自动提醒功能。就等于把煮水这个任务放到消息队列里处理。结束会自动告知用户（煮开提示音之类的）。用户只需要放水去煮就可以去干其他事情。这就是异步非阻塞AIO。

  4. 至于异步阻塞，那基本太傻了。
